# CLAUDE / AI Rules

## Project stack

- Node.js 24 LTS
- npm package manager
- Git commit conventions: Conventional Commits 1.0.0
- Editor: VS Code or equivalent

## Conventions

- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation changes
- `chore:` for tooling, config, and housekeeping
- `style:` for formatting or whitespace-only changes
- `refactor:` for code changes that do not add features or fix bugs

## AI Assistant guidance

- Prioritize README clarity, project purpose, and setup instructions.
- Keep commit messages Conventional Commit style.
- Suggest improvements to documentation and workflows.
- Respect the minimal Node.js setup and avoid adding unnecessary dependencies.
- Use a straightforward, friendly tone in generated docs.
